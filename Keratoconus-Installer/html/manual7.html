Errors and logging

If the application encounters an unexpected situation, it will pop up a “warning” or an “error” dialogue; a “warning” indicates that the application can still function, while an “error” indicates that the application is unable to proceed further and will terminate. In both cases the dialogue text refers to a log file where more information can be found; if you submit an error report then you should include the most recent log file(s).

The log files are stored on the hard disc in directory %LOCALAPPDATA%\Keratoconus\logging, where %LOCALAPPDATA% refers to the “local AppData” directory. The precise location of this directory depends on the version of windows, for example on Windows 8 it is C:\Users\username\AppData\Local. If you do not know how to find this directory then ask a wizard to help you.

A new log file is started each time that the application is launched, and the name incorporates an ISO 8601 timestamp – thus for example a file called keratoconus-log-20150724T071440Z.txt will have been created on 24 July 2015 at 07:14:40 UTC. If the log files are no longer required you may safely delete them at any time.
